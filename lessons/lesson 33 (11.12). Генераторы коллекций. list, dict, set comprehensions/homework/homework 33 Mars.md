## Краткая сводка
- На занятии не зафиксировано решений, поэтому ДЗ закрывает ключевые блоки: list/dict/set comprehension и комбинированные структуры.
- Цель — научиться записывать фильтры и форматирование прямо внутри comprehension, избегая промежуточных переменных и побочных эффектов.

## Задача 1. Разогрев — «температуры недели»
- **Описание:** дан список `temperatures = [-3, 0, 4, -6, 7, 10, 2]`. Сформируй список строк `["День 1: -3°C", ...]` только для температур > -2. Нумерация дней начинается с 1.
- **Критерии:** один list comprehension, знак `+` добавляется вручную для положительных температур, исходный список не изменяется.

## Задача 2. Практика — «фильтр покупок»
- **Описание:** получен список заказов `orders = [("Анна", 32000, "Казань"), ("Игорь", 61000, "Тюмень"), ...]`. Построй список строк `"Имя купил(а) на SUM"`, где:
  - учитываются только города, начинающиеся на `К`;
  - суммы ≥ 50 000 выводятся как `61000K`, остальные — `32000₽`;
  - для имён на `а/я` используй «купила».
- **Критерии:** один list comprehension, условное окончание суммы и глагола реализованы в выражении, исходный список не меняется.

## Задача 3. Dict comprehension — «баланс отделов»
- **Описание:** список словарей:
  ```python
  departments = [
      {"name": "Support", "budget": 150000, "employees": [{"salary": 40000}, {"salary": 50000}]},
      ...
  ]
  ```
  - Построй словарь `dept_budget`, где ключ — название, значение — строка `"ok"` если сумма зарплат ≤ бюджета, иначе `"over"`.
  - Включай только отделы с ≥ 2 сотрудниками.
- **Критерии:** один dict comprehension, суммирование через генератор прямо внутри выражения, исходные данные не меняются.

## Задача 4. Set comprehension — «лог авторизаций»
- **Описание:** есть список строк `logs = ["alex:login", "bob:update", "claire:logout"]`. Построй множество пользователей, у которых действие `"login"` или `"update"`.
- **Критерии:** один set comprehension, проверка формата строки (2 части через `:`), исходный список не изменяется.
