# Задание 29.3. Коллекции уровня «джун+». списки, срезы, вложенные структуры

## Блок урока
Мутация и методы списков. Закрепляем добавление, вставку и удаление элементов, чтобы уверенно управлять списком без дублирования данных.

## Условие задачи
Ты составляешь полку чтения для зимнего марафона:
1. Создай список базовых книг по порядку прохождения.
2. Добавь в конец рекомендации от друга, используя `extend`.
3. Вставь в позицию №3 книгу, которую нужно прочитать раньше остальных.
4. Удали по значению книгу, которая уже есть в аудиоформате.
5. Перемести последний элемент на вторую позицию с помощью `pop` и `insert`.
6. Выведи итоговый список с нумерацией, чтобы свериться с планом.

## Пример входных данных
```python
reading_list = ["классика", "история игр", "повесть о путешествиях", "сборник эссе"]
friend_tips = ["техническая литература", "комиксы"]
```

## Пример выходных данных
```
1. классика
2. комиксы
3. история игр
4. повесть о путешествиях
5. сборник эссе
6. техническая литература
```

## Критерии успешного выполнения
- Применены минимум три разных метода изменения списка
- Перемещение элементов выполняется через `pop` и `insert`
- Все операции отражаются в финальном выводе
- В ходе решения используется только исходный список `reading_list`

## Подсказки
- После `extend` дополнительные книги сразу появятся в `reading_list`
- Индекс 2 вставляет элемент как третий по порядку
- Для вывода с нумерацией удобно использовать `enumerate(..., start=1)`

## Частые ошибки и ограничения
- Не удаляй элемент, если не уверен в его наличии
- Не меняй список внутри того же цикла, в котором его перебираешь
- Избегай ручного обмена значениями, используй методы списка
- Следи, чтобы индексы не выходили за пределы длины списка
