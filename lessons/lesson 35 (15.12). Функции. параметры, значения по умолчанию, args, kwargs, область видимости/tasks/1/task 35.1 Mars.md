# Практическое задание 35.1

Это задание относится к первому блоку темы "Функции". Вы уже изучили основы работы с коллекциями (списки, словари, множества) и алгоритмами. Теперь вы переходите к ключевому инструменту структурирования кода — функциям. В этом блоке вы научитесь объявлять функции, передавать в них данные через параметры и получать результат с помощью оператора `return`. Это фундамент для всех последующих тем, связанных с функциями.

## Задания

### Описание задачи
Ваш друг Марс увлекается астрономией и ведет список планет Солнечной системы, которые он уже изучил. Он хранит этот список в программе. Сейчас ему нужно написать функцию, которая будет добавлять новую планету в его список изученного и возвращать обновленный список.

Создайте функцию с именем `add_planet`. Она должна принимать два аргумента:
1.  `planet_list` — список строк (названий планет), который нужно обновить.
2.  `new_planet` — строка, название планеты для добавления.

Функция должна добавить новую планету в конец переданного списка `planet_list` и вернуть этот обновленный список. Если планета с таким названием уже есть в списке, функция должна вернуть список без изменений и вывести сообщение: `"Планета {название_планеты} уже изучена!"`.

**Важно:** Функция должна именно *возвращать* (`return`) список, а не просто печатать (`print`) его. Это позволит дальше использовать результат ее работы в других вычислениях.

### Пример входных и выходных данных

**Пример 1: Добавление новой планеты.**
```python
# Исходный список изученных планет
my_planets = ["Марс", "Венера", "Земля"]

# Вызов функции для добавления "Юпитер"
updated_list = add_planet(my_planets, "Юпитер")

print("Обновленный список:", updated_list)
print("Исходный список теперь:", my_planets)
```
**Вывод на экран:**
```
Обновленный список: ['Марс', 'Венера', 'Земля', 'Юпитер']
Исходный список теперь: ['Марс', 'Венера', 'Земля', 'Юпитер']
```
*Пояснение:* Функция получила список `my_planets` и строку `"Юпитер"`. Она добавила "Юпитер" в конец этого списка и вернула его. Обратите внимание, что исходный список `my_planets` тоже изменился, потому что в функцию был передан сам список, а не его копия.

**Пример 2: Попытка добавить уже изученную планету.**
```python
my_planets = ["Марс", "Венера", "Земля"]

# Пытаемся добавить "Марс", который уже есть в списке
result = add_planet(my_planets, "Марс")

print("Результат функции:", result)
```
**Вывод на экран:**
```
Планета Марс уже изучена!
Результат функции: ['Марс', 'Венера', 'Земля']
```
*Пояснение:* Функция проверила, есть ли `"Марс"` в списке `my_planets`. Поскольку планета найдена, функция вывела предупреждающее сообщение и вернула исходный список без изменений. Переменная `result` теперь содержит тот же список, что и `my_planets`.

### Критерии проверки и ограничения
1.  Функция должна быть объявлена с помощью `def` и иметь имя `add_planet`.
2.  Функция должна иметь ровно два параметра в указанном порядке: `planet_list`, `new_planet`.
3.  Для проверки наличия планеты в списке используйте оператор `in`.
4.  Для добавления элемента в список используйте метод `.append()`.
5.  Функция **обязательно** должна использовать оператор `return` для возврата списка.
6.  Сообщение о дубликате планеты выводится с помощью `print`.
7.  Не используйте темы из будущих блоков: значения по умолчанию, именованные аргументы, `*args`, `**kwargs`, `global`.

### Решение задачи (псевдокод)
1.  Объяви функцию `add_planet` с двумя параметрами: `planet_list` и `new_planet`.
2.  Внутри функции проверь, содержится ли `new_planet` в списке `planet_list`.
    *   Если содержится:
        *   Выведи сообщение: `"Планета {new_planet} уже изучена!"`.
        *   Верни список `planet_list` (без изменений).
    *   Если не содержится:
        *   Добавь `new_planet` в конец списка `planet_list`.
        *   Верни список `planet_list` (теперь он обновлен).

### Рекомендации
-   **Вынести в отдельную функцию:** В данном случае вся логика задачи — это и есть одна функция. Выносить что-то еще не требуется.
-   **PEP8:**
    -   Используйте осмысленные имена для функции и параметров, как в задании (`add_planet`, `planet_list`, `new_planet`). Это делает код понятным.
    -   После объявления функции `def add_planet(...):` оставьте две пустые строки перед следующим блоком кода (например, перед `my_planets = ...`).
