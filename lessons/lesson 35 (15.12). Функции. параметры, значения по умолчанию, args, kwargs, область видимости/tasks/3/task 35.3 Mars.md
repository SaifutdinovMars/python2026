# Практическое задание 35.3

В предыдущих блоках ты научился создавать функции с параметрами, передавать аргументы позиционно и по имени, а также задавать для них безопасные значения по умолчанию. Сейчас мы переходим к ключевой теме этого урока: созданию гибких функций, которые могут работать с переменным количеством данных. Ты освоишь `*args` для сбора позиционных аргументов в кортеж и `**kwargs` для сбора именованных аргументов в словарь, а также технику распаковки списков и словарей при вызове функций. Эти навыки позволят тебе писать универсальный код, способный обрабатывать заранее неизвестное количество входных данных.

## Задания

### Описание задачи
Ты разрабатываешь модуль для обработки данных о студентах. Данные могут поступать в разном формате и количестве. Твоя задача — написать функцию `create_student_profile`, которая будет гибко формировать профиль студента.

Функция должна:
1.  Принимать обязательные аргументы: `name` (имя) и `student_id` (номер студенческого билета).
2.  Принимать любое количество дополнительных оценок за тесты через `*args`. Оценки передаются как позиционные аргументы после обязательных.
3.  Принимать любое количество дополнительных характеристик через `**kwargs`. Характеристики передаются как именованные аргументы (например, `hobby="шахматы"`, `city="Казань"`).
4.  Возвращать словарь профиля со следующей структурой:
    *   Ключи `"name"` и `"id"` с соответствующими значениями.
    *   Ключ `"test_scores"` со значением — кортежем из всех переданных оценок. Если оценки не переданы, кортеж должен быть пустым.
    *   Ключ `"attributes"` со значением — словарём всех переданных дополнительных характеристик.

Также напиши вторую функцию `print_profile`, которая будет красиво выводить профиль на экран. Она должна принимать словарь профиля и выводить данные в читаемом формате. Для передачи словаря профиля в эту функцию используй распаковку словаря (`**`).

### Пример входных и выходных данных

**Пример 1: Базовый вызов**
```python
# Вызов функции
profile1 = create_student_profile("Алексей", "STU-001", 85, 92, 78, city="Москва", course=2)
print_profile(profile1)
```
**Ожидаемый вывод:**
```
Студент: Алексей (ID: STU-001)
Оценки за тесты: (85, 92, 78)
Дополнительно: {'city': 'Москва', 'course': 2}
```
*Пояснение:* Аргументы `85, 92, 78` собрались в кортеж `args`. Аргументы `city="Москва", course=2` собрались в словарь `kwargs`.

**Пример 2: Вызов без дополнительных данных**
```python
# Вызов функции
profile2 = create_student_profile("Мария", "STU-002")
print_profile(profile2)
```
**Ожидаемый вывод:**
```
Студент: Мария (ID: STU-002)
Оценки за тесты: ()
Дополнительно: {}
```
*Пояснение:* Так как `*args` и `**kwargs` не получили аргументов, кортеж и словарь остались пустыми.

**Пример 3: Использование распаковки при вызове**
```python
# Данные хранятся в коллекциях
grades = [95, 88]
info = {"dormitory": "№5", "scholarship": True}

# Распаковываем список grades и словарь info при вызове
profile3 = create_student_profile("Иван", "STU-003", *grades, **info)
print_profile(profile3)
```
**Ожидаемый вывод:**
```
Студент: Иван (ID: STU-003)
Оценки за тесты: (95, 88)
Дополнительно: {'dormitory': '№5', 'scholarship': True}
```
*Пояснение:* Список `grades` был распакован в позиционные аргументы `95, 88`. Словарь `info` был распакован в именованные аргументы `dormitory="№5", scholarship=True`.

### Критерии проверки и ограничения
1.  Функция `create_student_profile` должна быть объявлена с параметрами `name`, `student_id`, `*args`, `**kwargs`.
2.  Внутри функции `args` должен быть преобразован в кортеж для хранения в итоговом словаре.
3.  Функция `print_profile` должна принимать словарь и использовать распаковку (`**profile_dict`) для передачи его содержимого в `print` или f-строку для формирования вывода.
4.  Код должен корректно обрабатывать случаи, когда `*args` или `**kwargs` не получают аргументов.
5.  **Важно:** В решении нельзя использовать темы из будущих блоков, такие как `global`, замыкания или обсуждение области видимости переменных. Работай только с передачей и распаковкой аргументов.

### Решение задачи (псевдокод)

**Для функции create_student_profile:**
1.  Объяви функцию с параметрами: имя, id, *args, **kwargs.
2.  Создай пустой словарь для профиля.
3.  Добавь в словарь пару "ключ-имя": значение-имя.
4.  Добавь в словарь пару "ключ-id": значение-id.
5.  Преобразуй все переданные позиционные аргументы (args) в кортеж и добавь его в словарь под ключом "test_scores".
6.  Добавь в словарь под ключом "attributes" словарь всех переданных именованных аргументов (kwargs).
7.  Верни получившийся словарь профиля.

**Для функции print_profile:**
1.  Объяви функцию, принимающую словарь профиля.
2.  Используя данные из словаря, собери и напечатай строку в формате:
    *   Строка с именем и ID.
    *   Строка с кортежем оценок.
    *   Строка со словарём дополнительных атрибутов.
3.  Для доступа к данным в f-строке используй распаковку словаря профиля.

### Рекомендации
-   Вынести в отдельную функцию: Задачу логично разделить на две функции: одну для создания структуры данных (`create_student_profile`) и одну для её отображения (`print_profile`). Это соответствует принципу единственной ответственности.
-   PEP8:
    -   Используй осмысленные имена для параметров, например, `student_id` вместо `id`, чтобы избежать конфликта со встроенной функцией `id()`.
    -   После блока импортов и перед объявлением функций оставляй две пустые строки.
