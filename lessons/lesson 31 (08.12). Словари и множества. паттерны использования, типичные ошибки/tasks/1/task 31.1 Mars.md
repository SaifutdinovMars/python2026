# Задание 31.1. Словари и множества. паттерны использования, типичные ошибки
## Блок урока
Словарь как структура быстрого доступа — Блок показывает, как превращать повторяющиеся данные в словарь и мгновенно находить значения по ключу.

## Условие задачи
Оранжерея хранит данные о сенсорах и последних поливах секций. Агроном проверяет, не пропущена ли секция, и сразу видит пробелы. Создай словарь last_watering, где ключ — код секции, а значение — время последнего полива. Затем выведи информацию для переменных sensor_to_check и unlinked_sensor.

## Пример входных данных
```python
sections = [
    ('A12', '2025-12-05 07:20'),
    ('B04', '2025-12-04 19:40'),
    ('C01', '2025-12-03 21:10'),
]
sensor_to_check = 'B04'
unlinked_sensor = 'D09'
```

## Пример выходных данных
```text
B04 -> 2025-12-04 19:40
D09 -> нет показаний
```

## Критерии успешного выполнения
- Список пар преобразован в словарь last_watering с ключом 'код секции'.
- Для обращения к словарю используется метод get с значением по умолчанию.
- Выведены ответы для sensor_to_check и unlinked_sensor в заданном формате.
- Код не падает при запросе отсутствующего ключа.

## Подсказки
- Пройдись по списку sections в цикле и наполняй словарь last_watering.
- Используй множественное присваивание: key, value = запись из sections.
- Для отсутствующих данных возвращай строку 'нет показаний'.

## Частые ошибки и ограничения
- Не обращайся к словарю через квадратные скобки, если нет проверки на существование ключа.
- Не оставляй ключи кортежами целиком — нужен именно идентификатор.
- Не дублируй вывод: достаточно одной строки на каждый запрос.
